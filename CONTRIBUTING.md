
# Guia de Contribuição

Este documento descreve o processo de contribuição para o projeto de Biblioteca Digital.

## Fluxo de Trabalho

1. Realize alterações no código.
2. Teste as funcionalidades modificadas.
3. Registre as alterações utilizando commits descritivos.
4. Envie as alterações para o GitHub.
5. Abra um Pull Request para revisão.

---

## Commit

Após realizar alterações:

git add .

git commit -m "Descrição clara da alteração"

Exemplo:

git commit -m "Adiciona organização de documentos por tipo e ano"

---

## Push

Para enviar as alterações ao GitHub:

git push origin main

---

## Pull Request

1. Acesse o repositório no GitHub.
2. Selecione a opção "Pull Requests".
3. Clique em "New Pull Request".
4. Descreva as alterações realizadas.
5. Solicite revisão.
6. Realize o merge após aprovação.

---

## Boas Práticas

- Utilizar mensagens de commit claras.
- Testar o sistema antes de enviar alterações.
- Documentar novas funcionalidades.
- Manter o código organizado e modularizado.
